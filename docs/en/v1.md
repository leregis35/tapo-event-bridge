# Version 1.0

Version 1.0 marks the first stable Tapo Event Bridge contract.

## Stability guarantees

- Dashboard payload schema version 1 remains backward compatible throughout 1.x.
- Events are retained only in a bounded memory buffer.
- Duplicate state transitions can be suppressed without camera polling.
- Discovery reads Home Assistant registries only and does not wake battery cameras.

## New diagnostics

The **Runtime metrics** sensor exposes uptime, buffer size, duplicate suppression,
listener count, cleanup callbacks and active transports. The **Clear event history**
button clears only the integration's in-memory buffer; it does not delete Home
Assistant recorder history.
